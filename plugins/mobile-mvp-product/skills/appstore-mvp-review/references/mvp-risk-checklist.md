# MVP risk checklist

Use this reference when reviewing a mobile app MVP before release.

## Common product risks

- The app is only one prompt/result screen
- It looks too similar to a competitor
- Onboarding promises too much
- Paywall appears before any trust is built
- No saved history, progress, or personalization
- No real empty/loading/error states
- Too many planned features, not enough finished core flow

## Common review-readiness improvements

- Add a visible core workflow, not only an AI answer
- Add history, plans, rooms, saved items, progress, streaks, or another persistent user layer
- Add real examples and realistic copy
- Add clear support/privacy/settings surfaces
- Add careful AI uncertainty wording
- Make screenshots match real in-app behavior

## MVP release rule

Ship when the core loop is complete, understandable, and useful. Do not wait for every secondary feature.
