# Mobile MVP UI patterns

Use this reference when creating mobile app screens from a product idea.

## Recommended first version structure

- Onboarding: 2-4 screens maximum, focused on value and setup
- Home: one clear next action, recent activity, and progress/history if relevant
- Core action: scan, record, practice, search, create, or start task
- Result/detail: useful output with next step, save/share/retry where relevant
- Paywall: value-based, not only feature-list-based
- Settings: account, language, subscription, support, privacy

## Screen quality rules

- One primary action per screen
- Avoid more than two hierarchy levels in cards
- Avoid huge text walls
- Use real-world copy, not lorem ipsum
- Use states: loading, empty, error, permission denied
- Make the app's unique angle visible above the fold

## Good MVP UI signals

- Real examples instead of placeholders
- User progress or saved state
- Obvious navigation
- Trust-building microcopy
- Small moments of personalization
- Practical constraints that match the platform

## Paywall rules

- Show premium value in the user's language
- Use 3-5 benefit bullets maximum
- Include restore purchase
- Include legal/subscription footer placeholder
- Do not block first useful moment unless the business model requires it
