# Contributing

Thanks for improving these Claude Code skills.

## Principles

- Keep `SKILL.md` focused and readable.
- Put detailed optional material in `references/`.
- Make descriptions specific, because Claude uses them to decide when a skill should trigger.
- Prefer practical mobile product guidance over generic AI instructions.
- Avoid executable scripts unless there is a strong reason.

## Pull request checklist

- [ ] Skill description still clearly explains when to use the skill
- [ ] Instructions are specific and actionable
- [ ] No secrets, tokens, or private URLs are included
- [ ] Version number bumped if behavior changed significantly
- [ ] README updated if install/use behavior changed
